# Discovery Question Bank

Use these as a reference. Ask one question at a time. Don't run through the list mechanically - pick the questions most relevant to the idea being explored.

## Problem understanding

- What problem does this solve? For whom specifically?
- How do users currently deal with this problem? (workarounds, competing tools, manual steps)
- How often does this problem occur? Daily, weekly, occasionally?
- How painful is it on a scale of "annoying" to "blocking"?
- Who feels this pain most acutely - a specific role, team size, or use case?
- Is this a problem users have told you about, or one you've inferred?

## Scope and success

- What does success look like - what would be measurably different?
- What's the smallest version of this that would be valuable?
- What would make this a failure even if it shipped?
- Is there a version of this that's already been tried? What happened?

## Market and viability

- Are competitors solving this? How?
- Would users pay for this, or is it table stakes?
- Does this help us win new users, retain existing ones, or expand revenue?
- What's the business case - cost reduction, revenue growth, retention, compliance?

## Feasibility

- Do we have the data we need to build this?
- Does this require new infrastructure or can it use what we have?
- Are there regulatory or compliance constraints?
- What's the riskiest technical assumption?

## Usability

- How would a user discover this feature?
- What's the expected learning curve?
- Could this be misused or misunderstood? How?
- What does the happy path look like in one sentence?

## Prioritisation signals

- Why this? What's changed that makes this the right opportunity?
- What would we have to deprioritise to pursue this?
- What's the cost of NOT building this?
- What decision does this unblock for users?
